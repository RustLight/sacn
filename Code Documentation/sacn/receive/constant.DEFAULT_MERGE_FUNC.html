const DEFAULT_MERGE_FUNC: fn(_: &DMXData, _: &DMXData) -> Result<DMXData>

If a packet for a universe is waiting to be synchronised and then another packet is received with the same universe and synchronisation address this situation must be handled. By default the implementation discards the lowest priority packet and if equal priority it discards the oldest packet as per ANSI E1.31-2018 Section 6.2.3.

This can be changed by providing a new function to handle the situation of the user implementing a custom merge/arbitration algorithm as per ANSI E1.31-2018 Section 6.2.3.2.
